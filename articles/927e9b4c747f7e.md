# AWSでポートフォリオサイトを構築する：S3 + CloudFront + ACM + Cloudflare

## はじめに

この記事では、独自ドメインを使ったポートフォリオサイトをAWSで構築した手順をまとめます。

**完成した構成：**

```
exobrainlab.com
    ↓ (Cloudflare DNS → CNAME)
CloudFront (CDN + HTTPS)
    ↓ (OAC)
S3 (静的サイトホスティング)
        ↑
ACM (SSL証明書 / us-east-1)
```

**使用サービス：**
- AWS ACM（SSL/TLS証明書）
- AWS S3（静的ファイルホスティング）
- AWS CloudFront（CDN）
- Cloudflare（ドメインレジストラ + DNS）

**コスト：**
- ACM：無料
- S3：ほぼ無料（個人サイトレベル）
- CloudFront：ほぼ無料（個人サイトレベル）
- Cloudflare：無料プラン

---

## 前提条件

- AWSアカウント取得済み
- Cloudflareでドメイン取得済み（本記事では `exobrainlab.com`）

---

## Step 1：ACMでSSL証明書を発行する

:::message alert
ACMは必ず **バージニア北部（us-east-1）** リージョンで作成してください。CloudFrontはus-east-1のACM証明書しか使えません。
:::

### 1-1. AWSコンソールでリージョンを切り替える

AWSコンソール右上のリージョン選択で「米国東部（バージニア北部）us-east-1」を選択します。

![リージョン切り替え](screenshot_region.png)

### 1-2. ACMを開いて証明書をリクエスト

検索バーで「ACM」を検索 → Certificate Managerを開く → 「証明書をリクエスト」をクリック。

![ACMトップ画面](screenshot_acm_top.png)

### 1-3. ドメイン名を入力

「完全修飾ドメイン名」に以下の2つを入力します。

```
exobrainlab.com
*.exobrainlab.com
```

`*.exobrainlab.com` はワイルドカード証明書で、サブドメイン（www等）もカバーできます。

**その他の設定：**
- 検証方法：DNS検証（推奨）
- キーアルゴリズム：RSA 2048

![証明書リクエスト画面](screenshot_acm_request.png)

### 1-4. CNAMEレコードをエクスポートする

リクエスト完了後、「CSVにエクスポート」をクリックしてCNAMEレコードをダウンロードします。

![証明書詳細画面](screenshot_acm_detail.png)

CSVの中身はこのような形です：

```
ドメイン名,CNAME 名,タイプ,CNAME 値
exobrainlab.com,_678be728d57cddfaf3b9463779e56bfb.exobrainlab.com.,CNAME,_e3aa67a6704effbd39ea834239fa759a.jkddzztszm.acm-validations.aws.
```

---

## Step 2：CloudflareにACM検証用CNAMEを追加する

Cloudflareのダッシュボードで `exobrainlab.com` → DNS → Records を開き、「Add record」をクリックします。

| 項目 | 値 |
|------|-----|
| Type | CNAME |
| Name | `_678be728d57cddfaf3b9463779e56bfb`（CSVのCNAME名からドメイン部分を除いたもの） |
| Target | `_e3aa67a6704effbd39ea834239fa759a.jkddzztszm.acm-validations.aws.` |
| Proxy status | **DNS only（グレー）← 重要** |

:::message alert
Proxy statusは必ず **DNS only（グレーの雲）** にしてください。オレンジのままだとACM検証が通りません。
:::

![CloudflareのCNAME追加](screenshot_cloudflare_cname.png)

数分〜30分でACMのステータスが「発行済み」に変わります。

---

## Step 3：S3バケットを作成する

### 3-1. バケットを作成

S3を開いて「バケットを作成」をクリックします。

**設定：**
- バケット名：`exobrainlab-website`（任意）
- リージョン：任意（今回はap-southeast-2）
- ACL：無効（推奨）
- パブリックアクセス：すべてブロック（そのまま）

:::message
CloudFront + OACの構成ではバケット名をドメインと一致させる必要はありません。
:::

![S3バケット作成](screenshot_s3_create.png)

---

## Step 4：CloudFrontディストリビューションを作成する

### 4-1. ディストリビューションを作成

CloudFrontを開いて「ディストリビューションを作成」をクリックします。

**Step 1（Get started）：**
- Distribution name：`exobrainlab-website`
- Distribution type：Single website or app
- Domain（Route 53 managed domain）：空欄のまま

![CloudFront Get started](screenshot_cf_step1.png)

**Step 2（Specify origin）：**
- Origin type：Amazon S3
- S3 origin：「Browse S3」で `exobrainlab-website` を選択
- Allow private S3 bucket access to CloudFront：チェックあり（OAC設定）
- Use recommended origin settings：選択

![CloudFront Specify origin](screenshot_cf_step2.png)

**Step 3（Enable security）：**
- WAF：「セキュリティ保護を有効にしないでください」を選択（月$14かかるため）

**Step 4（Review and create）：**
内容を確認して「Create distribution」をクリック。

### 4-2. 代替ドメインとSSL証明書を設定する

作成完了後、「Add domain」をクリックします。

**Configure domains：**
```
exobrainlab.com
www.exobrainlab.com
```

**Get TLS certificate：**
ACMで発行した証明書が自動で表示されるので選択します。

![CloudFront Add domain](screenshot_cf_domain.png)

### 4-3. デフォルトルートオブジェクトを設定する

「編集」をクリックして「Default root object」に `index.html` を入力して保存します。

![CloudFront 編集画面](screenshot_cf_edit.png)

:::message alert
この設定がないと `https://exobrainlab.com/` にアクセスしてもサイトが表示されません。
:::

---

## Step 5：CloudflareのDNSをCloudFrontに向ける

CloudflareのDNS Recordsで以下の2つのCNAMEレコードを追加します。

**ルートドメイン用：**

| 項目 | 値 |
|------|-----|
| Type | CNAME |
| Name | `@` |
| Target | `d3nu0979n5fl7d.cloudfront.net`（自分のCloudFrontドメイン） |
| Proxy status | **DNS only** |

**www用：**

| 項目 | 値 |
|------|-----|
| Type | CNAME |
| Name | `www` |
| Target | `d3nu0979n5fl7d.cloudfront.net` |
| Proxy status | **DNS only** |

![CloudflareのDNSレコード一覧](screenshot_cloudflare_dns.png)

---

## Step 6：index.htmlをS3にアップロードする

S3の `exobrainlab-website` バケットを開いて「アップロード」をクリックし、以下のHTMLファイルをアップロードします。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>Exobrain Lab</title>
</head>
<body>
  <h1>Exobrain Lab</h1>
  <p>Coming soon.</p>
</body>
</html>
```

![S3アップロード完了](screenshot_s3_upload.png)

---

## Step 7：動作確認

ブラウザで `https://exobrainlab.com` にアクセスして表示を確認します。

![サイト表示確認](screenshot_site.png)

「Exobrain Lab / Coming soon.」が表示されれば完成です🎉

DNSの伝播には数分かかる場合があります。表示されない場合は少し待ってから再度アクセスしてみてください。

---

## まとめ

今回構築した構成のポイントをまとめます。

**セキュリティ面：**
- S3バケットはパブリックアクセスをすべてブロック
- CloudFrontのOAC（Origin Access Control）でS3へのアクセスをCloudFrontのみに限定
- ACMで無料のSSL証明書を発行してHTTPS化

**コスト面：**
- CloudFrontとS3は個人サイトレベルではほぼ無料
- ACMはCloudFrontと組み合わせる場合は無料
- Cloudflareは無料プランで十分

**学習面：**
この構成はAWS SAA-C03（ソリューションアーキテクト アソシエイト）の試験範囲とも完全に一致しており、実際に手を動かして構築することで理解が深まります。
